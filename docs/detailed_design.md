## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

1. 起動処理
   - products.csv, coins.csv を読み込む
   - 読込に失敗した場合は管理者向けメッセージを表示して空データで続行するか終了する選択肢を提示。

2. メインメニューを表示する
   - 表示項目（例）:
     1. 会計開始（顧客モード）
     2. 取引ログ表示
     3. 管理者メニュー
     9. 終了

3. ユーザーの入力を受け取る（scanf）
   - 入力は整数（メニュー番号）で受け取る。scanf 失敗時は入力バッファをクリアして再表示。

4. 入力に応じて分岐する
   - 1 の場合: 会計処理（購入処理ルーチン）を呼ぶ
   - 2 の場合: transactions.csv を読み出してログ表示ルーチンを呼ぶ
   - 3 の場合: 管理者認証（PIN） → 管理者メニュー（商品追加/編集、釣銭補充 等）
   - 9 の場合: save state（必要なら coins.csv を保存）してプログラムを終了する
   - その他: エラーメッセージを表示してメニューを再表示

5. 処理が完了したらメインメニューに戻る（ループ）
   - 常に状態を保存する（商品編集や釣銭補充があれば対応ファイルへ書き込み）

（擬似フロー）
while (true) {
  show_main_menu();
  if (!scanf("%d", &choice)) { clear_input(); continue; }
  switch(choice) {
    case 1: handle_sale(); break;
    case 2: show_transactions(); break;
    case 3: if (verify_pin()) admin_menu(); break;
    case 9: persist_state(); exit;
    default: printf("無効な選択\n");
  }
}

### 4.2 個別機能のアルゴリズム

#### 機能1: 購入処理（会計フロー）
1. カート初期化（空の CartItem[]）
2. 商品追加ループ
   - ユーザーに商品ID（4桁）またはバーコードを入力させる
   - find_product_by_id(id) を呼ぶ
   - 見つからなければ「商品未登録」のメッセージを出し再入力（または商品検索／キャンセル）
   - 見つかったら数量を入力して add_to_cart(product, quantity)
   - 「続けて入力するか、会計へ進むか、カートを編集するか」を選ばせる
   - カート編集: 個別アイテムの数量変更／削除

3. 会計開始（checkout）
   - 小計を計算（各 CartItem の price * quantity の合計）
   - 税率別集計を行う:
     - 税率ごとに課税対象額、税額（切り捨て/四捨五入の方針を決める）を算出
     - 例: 税額 = floor(課税対象額 * 税率 / 100 + 0.5) （端数処理ポリシーを決定）
   - 合計（小計 + 全税額）を表示
   - 支払方法選択（現金 / カード（疑似） / 電子（疑似））

4. 支払処理（現金の場合）
   - 必要金額を表示し、投入口からの投入額（整数円）を逐次受け取る
   - 投入額 >= 合計 になったらおつりを計算
   - 釣銭算出アルゴリズム（次節参照）
   - 釣銭が用意できない場合は次の処理:
     - ユーザーに「釣銭不足のため現金では支払えません。カード/電子/現金で正確な金額を投入してください」と促す
     - 支払を中断し、投入金を返却する（疑似）
   - 支払成功時は int allcoins の在庫を更新（受け取った紙幣/硬貨のカウント増加、払い出した釣銭のカウント減少）

5. 支払処理（カード/電子）
   - 疑似承認（常に成功にする／簡易ランダムで成功失敗を返す）
   - 成功なら取引確定、失敗なら支払方法選択へ戻す

6. 取引確定後
   - レシート情報を画面表示
   - transactions.csv に追記（save_transaction）
   - カートをクリアしてメインメニューへ戻る

（釣銭算出：貪欲法＋在庫考慮）
- 利用可能な硬貨/紙幣の額面を降順に並べ（1000, 500, 100, 50, 10, 5, 1 等）
- 各額面について、必要枚数 = min(残りのおつり / 額面, 在庫枚数)
- 残りのおつりが0になるまで繰り返す
- 最終的に残りが0なら成功、残り>0なら釣銭不足（失敗）
- 必要ならバックトラック（単純な貪欲で失敗するケースに備え）を検討するが、まずは一般的な日本通貨の額面では貪欲で十分

#### 機能2: ログ保存（save_transaction）
1. トランザクション情報を整形
   - datetime（YYYY-MM-DD HH:MM:SS）
   - 商品名と数量（複数アイテムならセパレータで結合）
   - 小計、税額（税率別）、合計、支払方法
2. transactions.csv を append モードで開く
3. 1行分を CSV フォーマットで書き込む（ヘッダに合わせる）
4. ファイル書き込みエラーが発生したら、画面で管理者へ知らせる（ディスクフル等）

CSV 例行:
"2026-05-19 12:34:56","Apple x2; Banana x1",3,360,"cash",3600

（実装メモ）
- 同時実行がない学習用途ではファイルロックは省略可。ただし実運用ではロック／トランザクション制御を追加すること。

#### 機能3: ログ表示（show_transactions）
1. transactions.csv を開く
2. 各行を read してフィールド分解
3. 指定件数（最新 N 件）だけ表示できるオプションを用意
4. 表示できない／ファイルがない場合は「ログなし」メッセージ

#### 機能4: 管理者機能
- PIN 認証:
  1. PIN 入力（scanf）
  2. 照合（平文比較。実運用ではハッシュ化を推奨）
  3. 認証失敗は回数制限（例: 3 回）を設けてメニューに戻す
- メニュー:
  - 商品追加: products.csv に新行を追加（ID 重複チェック）
  - 商品編集: ID 指定で該当行を修正（読み込み→編集→上書き）
  - 釣銭補充: coins.csv の各額面を加算して保存
  - 取引集計: transactions.csv を読み込み、簡易レポート（総売上、件数、税率別合計）を作成

### 4.3 エラーハンドリング（想定される異常系）

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 数字以外の文字が入力された場合（メニュー/数量/金額） | scanf が失敗したら入力バッファをクリアし、エラーメッセージを表示して再入力を促す。最大再試行回数を設ける（例: 3 回）。 |
| 商品IDが未登録の場合 | 「商品が見つかりません」と表示し再入力を促す。検索／キャンセルの選択肢を提供。管理者なら商品登録へ誘導。 |
| CSV読込失敗（products.csv / coins.csv） | 起動時に致命的エラー扱いとして管理者に通知。空データで続行する選択肢を提示し、管理者メニューでデータ作成を促す。ログ保存失敗は画面にエラー表示し、メモリ上に保持して次回起動時に再試行する（簡易）。 |
| 釣銭不足 | 現金支払時に釣銭を用意できない場合は支払を拒否し、ユーザーに別決済手段を促す。管理者へは釣銭補充を依頼する。 |
| scanf 失敗（EOF/フォーマット不一致） | 入力バッファを読み捨て（例: while(getchar()!='\n');）し再入力を促す。繰り返し発生する場合はメニューへ戻す。 |
| ファイル書き込み失敗（transactions.csv append 失敗等） | エラー表示後、再試行をユーザーに促す。再試行しても失敗する場合は画面に保存できなかった旨を表示し、ローカル変数に未保存トランザクションを保持して管理者が回収できるようにする。 |
| 管理者PIN漏洩（学習用途以外） | ドキュメントで平文保存は不可と明記。運用時はハッシュ化・アクセス制御を行う。 |
| 不正な数量（負数、大きすぎる数） | 入力時に範囲チェックを行う（例: 1〜999）。範囲外はエラー表示して再入力。 |

（入力検証ルール例）
- 数量: int, 1 <= quantity <= 999
- 金額投入: int >= 0
- メニュー: 有効な番号のみ受け付け

---

実装上の注意点（短く）
- 通貨や端数処理のルールは設計段階で厳密に決める（税額の丸め方法など）。
- CSV 操作は行単位での読み込み・分解（strtok 等）を行う。大量データ時は配列サイズ上限（最大1500件）をチェックする。
- メモリ管理に注意（文字列のコピー、バッファサイズ超過を防ぐ）。
- scanf 使用時はバッファオーバーフローを防ぐためフォーマット指定子に幅指定を入れる（例: "%4s"、"%99s"）。
- 単体テスト項目：CSV 読込、find_product_by_id、add_to_cart、tax_calc、change_calc、save_transaction。
