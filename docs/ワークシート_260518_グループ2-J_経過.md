## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューを表示する
2. ユーザーの入力を受け取る
3. 入力に応じて分岐する
   - 1の場合: （処理を記述）
   - 2の場合: （処理を記述）
   - 9の場合: プログラムを終了する
4. メインメニューに戻る（1へ）

【スタート画面】
１．画面をタッチしてスタート
   →print関数で表示
２．会計が終わるとスタート画面に戻る。
   →戻り値を設定する。

【レジ袋】
１．print関数で「レジ袋を購入しますか」と表示
   →はいで料金を加算 いいえでそのまま

【商品スキャン】
１．商品をスキャン（コード入力）する。

【バーコードなし】
１．コードスキャンで見つからない場合はエラー表示する。
２．スタッフボタンで呼ぶ
３．Pass入力後手打ちで値の入力

【キャンセル選択】
１．個数を表示
２．あってない場合はキャンセルボタンを表示
３．問題なければ次へ

【年齢確認】
１．20歳ですかと確認
   →Yesで次へNoで決済へ
２．購入するものの判別（お酒orたばこ）
３．スキャンを続ける
４．スキャンできなかった場合エラー表示で店員を呼び出す。
５．買う物が確定なら次に進む

【決済選択】
１．支払方法の一覧を表示
   →現金、カード、QR
２．選択した画面の表示
３．現金の場合、投入金額をscanfで入力する。
   →足りた場合は次へ　不足の場合は「金額不足です。」と表示し、再入力
４．カード、QRの場合
   →足りていれば決済完了
   →残高不足の場合「金額不足ですと表示」し、ほかの支払い方法を提示
５，決済が完了したら決済完了画面を表示し、次に進む。

【レシート発行】
１．定義された変数から値を取り出す。
２．printfでターミナルに表示
    →確認しやすいように改行する。
    例）2026/05/18 11:30
        りんご  ５個    ￥５５０
        バナナ  ７個    ￥７７０
        ５０円引き      ￥-５０

        お会計          ￥１２５０
                　（消費税１０％）
３．CSVファイルにデータを保存
    例）2026/02/15, 16:00, りんご, 1, 100, 10%, -50, (100-50)-10%
４．値が取り出せなかった場合にエラーを出す。
５．次の手順に進む。

【釣銭チェック】
１．お金を管理する変数を定義する。
    →電子決済は別の変数
２．上限値を設定する（仮で１０万）
３．会計で現金を選択した場合のみ上限まで加算する。
４．上限-5,000円でエラー表示を出す。
５．エラー時はPassを使って内部のお金を取り出す。
６．エラーがなければ次の手順に進む
```
