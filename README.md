本ディレクトリは、皆様のチームが4日間（Day1〜Day4）かけて作り上げる**「キオスク端末シミュレータ」**の開発拠点（ワークスペース）です。
前半2日間（Day1〜2）は**手作業**で設計・実装を行い、後半2日間（Day3〜4）で**AIツールを解禁**して開発を進めます。

## 📂 開発ルールとディレクトリ構成

研修中のソースコード（`.c` ファイル）は、整理のために **`src/`** フォルダの中で作成・編集することを推奨します。

```text
├── README.md        ← 今見ているこのファイル・チームの看板
├── ワークシート.md   ← 設計ワークシート（Day1〜2で記入）
├── build.bat        ← コンパイル用スクリプト
├── .vscode/         ← VSCodeの設定（ビルドタスク等）
├── src/             ← ソースコードはこの中に入れる
│   └── main.c       ← メインのプログラム
└── data/            ← CSVデータの保存先
```

## 🔨 ビルド（コンパイル）と実行

### 方法1: VSCode（推奨）

1. `Ctrl + Shift + B` を押す → 自動的にコンパイルされます
2. ターミナルに `app.exe` が表示されたら成功
3. ターミナルで `.\app.exe` と入力して実行

### 方法2: ターミナルで直接実行

```
gcc src\main.c -o app.exe -Wall
.\app.exe
```

### 方法3: build.bat

```
.\build.bat
.\app.exe
```

---

## ⚠️ Gitコミット時の注意
*   コンパイル後の実行ファイル（`app.exe` 等）は、Gitなどのバージョン管理には含めない（コミットしない）のが商用開発の鉄則です。
*   あらかじめ `.gitignore` にはプロ仕様の除外設定が書かれているため、そのまま `git add .` しても安全なように守られています。
*   （※）顧客の個人情報に見立てたログ記録ファイル（`*.csv`）も、Git上にアップロードされないよう制限をかけてあります。
